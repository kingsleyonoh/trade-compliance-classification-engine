# PR Checklist

Before merging into `dev`:

- [ ] PR references a `docs/progress.md` task.
- [ ] Active claim exists under `docs/claims/` or the PR explains why no claim was needed.
- [ ] Tests passed.
- [ ] E2E evidence included for API/UI/JOB/INTEGRATION changes.
- [ ] Secret scan passed.
- [ ] No protected governance files changed unless explicitly intended.
- [ ] No overlap with another active claim's expected files.
- [ ] Documentation or build journal updated when behavior changed.
