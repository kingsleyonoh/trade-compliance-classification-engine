# PR Checklist

Before merging into `dev`:

- [ ] PR references an issue/task, or explains why no external task was needed.
- [ ] Active private claim exists when your team uses claim files, or the PR explains why no claim was needed.
- [ ] Tests passed.
- [ ] E2E evidence included for API/UI/JOB/INTEGRATION changes.
- [ ] Secret scan passed.
- [ ] No protected governance files changed unless explicitly intended.
- [ ] No overlap with another active claim's expected files.
- [ ] Documentation updated when behavior changed.
